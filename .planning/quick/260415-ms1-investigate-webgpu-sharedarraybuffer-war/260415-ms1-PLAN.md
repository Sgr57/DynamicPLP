<objective>
Fix the "Questo dispositivo non supporta WebGPU o SharedArrayBuffer" warning on Vercel production by adding cross-origin isolation headers via vercel.json and aligning the Vite dev server to use the same `credentialless` COEP policy.

Purpose: Without COOP/COEP headers, `SharedArrayBuffer` is undefined in the browser on Vercel production. This blocks ONNX Runtime multi-threading (WASM fallback) and triggers a false device capability warning, making the app appear broken on production even though WebGPU is available.

Output: `vercel.json` at project root, updated `vite.config.js` with `credentialless` policy.
</objective>

Key details (per CONTEXT.md locked decisions):
- COEP value MUST be `credentialless` (not `require-corp`) — this allows cross-origin fetches to HuggingFace CDN for model files without requiring the remote server to send CORP headers.
- Source pattern `/(.*)`matches all routes including root — required because this is a SPA and all routes need SharedArrayBuffer for LLM inference.
- The `$schema` field enables editor autocomplete and validation for vercel.json.

Do NOT add any other Vercel configuration (rewrites, redirects, build settings, etc.) — only headers.
